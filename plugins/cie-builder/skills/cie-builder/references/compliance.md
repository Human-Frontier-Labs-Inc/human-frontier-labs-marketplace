# CIE Compliance Reference — TX-RAMP Level 1 + CJIS

## Overview

CIE targets TX-RAMP Level 1 certification (117 controls across 17 families) while satisfying CJIS Security Policy requirements. This document maps each control family to concrete implementation patterns.

Many controls are organizational/procedural (policies, training, personnel). This reference focuses on the TECHNICAL controls that must be implemented in code and infrastructure. Procedural controls are flagged for documentation during Phase 6.

## Control Family Implementation Map

| Control | Name | Implementation | Phase |
|---------|------|---------------|-------|
| AC-01 | Policy and Procedures | Document in SSP | 6 |
| AC-02 | Account Management | User CRUD, role assignment, account lifecycle, annual review | 0 |

| AC-07 | Unsuccessful Logon Attempts | Max 5 attempts / 15 min lockout | 0 |
| AC-14 | Permitted Actions Without Auth | Explicitly enumerate: health check only | 0 |
| AC-17 | Remote Access | TLS everywhere, VPN for admin access | 0 |
| AC-18 | Wireless Access | GCP managed — document in SSP | 6 |
| AC-19 | Mobile Device Access Control | If mobile app: MDM requirements | 5+ |
| AC-20 | Use of External Systems | Document approved integrations | 6 |

**CJIS Addition**: Advanced Authentication (AA) required for all access to CJI data. MFA is mandatory, not optional.

#### Implementation Patterns

```go
// Row-level security — EVERY query must include org_id
func (r *CallEventRepo) GetByID(ctx context.Context, orgID, callID string) (*CallEvent, error) {
    query := `SELECT * FROM call_events WHERE id = $1 AND org_id = $2`
    // org_id comes from JWT, never from request body
    return r.db.QueryRow(ctx, query, callID, orgID)
}

// Failed login tracking
type LoginAttempt struct {
    UserID    string
    IP        string
    Timestamp time.Time
    Success   bool
}
// Lock after 5 failures in 15 minutes
```

```go
type AuditEntry struct {
    ID          string    `json:"id"`
    Timestamp   time.Time `json:"timestamp"`   // UTC always

    UserID      string    `json:"user_id"`

    UserAgent   string    `json:"user_agent"`

// Audit table: NO UPDATE, NO DELETE permissions granted
```

### Identification and Authentication (IA) — 10 Controls

| Control | Name | Implementation | Phase |
|---------|------|---------------|-------|
| IA-01 | Policy and Procedures | Document in SSP | 6 |
| IA-02 | ID and Auth (Org Users) | Username + password + MFA | 0 |
| IA-02-01 | MFA for Privileged Accounts | TOTP required for Admin, Supervisor | 0 |
| IA-04 | Identifier Management | Unique user IDs, no shared accounts | 0 |
| IA-05 | Authenticator Management | Password policy: 12+ chars, complexity, 60-day rotation | 0 |
| IA-05-01 | Password-based Auth | No password hints, hashed with bcrypt (cost 12+) | 0 |
| IA-06 | Authentication Feedback | Obscure which field failed ("invalid credentials") | 0 |
| IA-07 | Cryptographic Module Auth | FIPS 140-2 validated modules for crypto operations | 0 |
| IA-08 | ID and Auth (Non-Org Users) | API keys for system integrations, same audit trail | 1+ |
| IA-11 | Re-authentication | Required for: export, admin actions, after 30 min idle | 0 |

**CJIS Addition**: AA is mandatory. Passwords minimum 8 characters (we exceed at 12). MFA for ALL users accessing CJI, not just privileged.

#### Implementation Patterns

```go
// Password policy enforcement
type PasswordPolicy struct {
    MinLength       int  // 12
    RequireUpper    bool // true
    RequireLower    bool // true
    RequireDigit    bool // true
    RequireSpecial  bool // true
    MaxAgeDays      int  // 60
    HistoryCount    int  // 12 (can't reuse last 12)
}

// Authentication feedback — NEVER reveal which field failed
// BAD:  "User not found"
// BAD:  "Incorrect password"
// GOOD: "Invalid credentials"
```

#### Implementation Patterns

```bash
# GCP encryption at rest — enable customer-managed keys
gcloud kms keyrings create cie-keyring --location=us-central1
gcloud kms keys create cie-db-key --keyring=cie-keyring --location=us-central1 --purpose=encryption

# Cloud SQL with CMEK
gcloud sql instances create cie-db --database-flags=cloudsql.iam_authentication=on \
  --disk-encryption-key=projects/PROJECT/locations/us-central1/keyRings/cie-keyring/cryptoKeys/cie-db-key

# TLS 1.2+ enforced
# In Go: tls.Config{MinVersion: tls.VersionTLS12}
```

## CJIS-Specific Requirements Beyond TX-RAMP

| Requirement | Implementation |
|------------|---------------|
| Advanced Authentication | MFA for ALL users, not just privileged |
| Encryption of CJI at rest | CMEK via Cloud KMS, FIPS 140-2 |
| Encryption of CJI in transit | TLS 1.2+ minimum |
| Audit of CJI access | Every read/write to correctional data logged |
| Personnel security | Background checks for all with CJI access |
| Physical security | GCP SOC 2 / FedRAMP covers physical |
| Incident response | 24-hour reporting requirement for CJI breaches |
| Session management | 30-minute inactivity timeout |
